## 1. 프로젝트 목적

본 프로젝트는 Java 기반 물류 관리 시스템으로,
입·출고, 재고, 주문, 배송 상태를 체계적으로 관리하는 것을 목표로 합니다.

실무에서 자주 사용되는 물류 도메인을 중심으로
도메인 설계, REST API, 트랜잭션 처리, 동시성 제어를 학습·적용하는 것을 주요 목적으로 합니다.


---

## 2. 도메인 개요

### 핵심 흐름

상품 등록
→ 창고 보관
→ 재고 입고
→ 주문 생성
→ 재고 차감
→ 출고 / 배송


## 3. 주요 기능
### 2.1 상품 관리

- 상품 등록 / 수정 / 삭제
- 상품 조회 (단건, 목록)
- 상품 상태 관리 (판매중, 품절 등)

### 2.2 재고 관리
- 창고별 재고 관리
- 입고 / 출고 처리
- 재고 수량 자동 증감
- 재고 부족 예외 처리

2.3 주문 관리
- 주문 생성
- 주문 상세 조회
- 주문 상태 관리
(CREATED → PAID → SHIPPED → DELIVERED → CANCELED)

2.4 배송 관리

- 배송 정보 등록

- 배송 상태 변경

- 주문과 배송 연계

2.5 공통 기능

- REST API 기반 설계

- 예외 처리 및 공통 응답 포맷

- 트랜잭션 처리

- 동시성 이슈 방지(재고 차감)
